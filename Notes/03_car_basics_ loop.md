# 第三讲：Python循环结构（while/for）+ 树莓派小车持续调速实战
> 核心目标：掌握两种循环的核心用法，结合条件判断实现小车重复调速、自动启停

## 一、循环的核心意义
循环是Python中「让代码重复执行」的核心语法，解决「重复写相同代码」的问题。
- 生活场景：让小车「重复加速」「重复接收指令」，不用写10遍加速代码；
- 技术价值：减少冗余代码，实现「自动化、持续性」的逻辑（如小车自动调速）。

## 二、while循环：按「条件」重复（未知次数）
### 1. 核心逻辑
「只要条件成立，就一直执行；条件不成立，立刻停止」→ 适用于「不知道要重复多少次」的场景。

### 2. 基础语法（小车场景化）
```python
# 初始化变量（小车初始速度）
current_speed = 20
# while + 判断条件：条件为True就重复执行缩进代码
while current_speed < 40:
    current_speed += 2  # 每次加速2km/h（必须修改判断变量，避免无限循环）
    print(f"当前速度：{current_speed} km/h")
# 条件为False时，跳出循环
print("已达目标速度，停止加速")
```

### 3. 关键特性
- ✅ 必须有「终止条件」（如`current_speed < 40`），否则会无限循环（小车卡死）；
- ✅ 循环内必须修改「判断变量」（如`current_speed += 2`），否则条件永远成立；
- ✅ 核心关键字`break`：强制终止整个循环（如超速保护）：
  ```python
  while current_speed < 40:
      current_speed += 2
      if current_speed >= 50:  # 超速保护
          print("⚠️ 即将超速，停止加速")
          break  # 立刻终止循环
  ```

### 4. 小车适用场景
- 持续加速/减速直到达到目标速度；
- 持续接收用户指令直到输入「停止」；
- 检测传感器数据，直到触发停止条件（如检测到障碍）。

## 三、for循环：按「次数」重复（已知次数）
### 1. 核心逻辑
「固定次数重复执行」→ 适用于「明确知道要做多少次」的场景（如减速5次、加速3次）。

### 2. 基础语法（小车场景化）
```python
current_speed = 40
# for + 变量 in range(次数)：重复执行N次（range(5)=执行5次）
for i in range(5):
    current_speed -= 2  # 每次减速2km/h
    # i+1：将「第0次」转为「第1次」，符合日常计数习惯
    print(f"第{i+1}次减速后速度：{current_speed} km/h")
```

### 3. 关键特性
- ✅ `range(N)`：生成0~N-1的数字，共执行N次（核心记：`range(5)`=5次）；
- ✅ 核心关键字`continue`：跳过本次循环，执行下一次（如低速保护）：
  ```python
  for i in range(5):
      current_speed -= 2
      if current_speed < 10:
          print("⚠️ 速度过低，跳过本次减速")
          current_speed += 2  # 恢复速度
          continue  # 跳过本次循环，不执行后续打印
      print(f"第{i+1}次减速后速度：{current_speed} km/h")
  ```

### 4. 小车适用场景
- 固定次数加速/减速（如减速5次、加速3次）；
- 遍历小车速度列表（如打印[10,20,30,40]所有速度）；
- 重复检测传感器数据N次（如检测5次障碍物）。

## 四、循环控制关键字：break vs continue
| 关键字 | 作用 | 小车场景示例 |
|--------|------|--------------|
| break  | 强制终止**整个循环**，后续代码不再执行 | 小车超速时，立刻停止加速循环 |
| continue | 跳过**本次循环**，直接执行下一次 | 小车速度过低时，跳过本次减速，继续下一次 |

## 五、综合实战：循环+条件判断 → 小车人机交互调速
### 1. 核心逻辑
「while循环（持续接收指令）+ if/elif/else（处理指令）」实现小车自动化控制。

### 2. 核心代码逻辑拆解
```python
current_speed = 15
car_running = True  # 小车运行状态：True=运行，False=停止

# while循环：持续接收指令直到停止
while car_running:
    user_cmd = input("请输入指令（加速/减速/停止）：")
    # 条件判断处理指令
    if user_cmd == "加速":
        current_speed += 5  # 加速逻辑
    elif user_cmd == "减速":
        current_speed -= 5  # 减速逻辑
    elif user_cmd == "停止":
        car_running = False  # 终止循环（小车停止）
    else:
        print("指令错误")
    # 边界检查（限速/低速保护）
    if current_speed > 50: current_speed = 50
    if current_speed < 0: current_speed = 0
```

### 3. 实战关键点
- `car_running`状态变量：控制while循环的启动/停止；
- 输入指令模拟：实际项目中可替换为「按键/传感器信号」；
- 边界检查：避免小车速度超标或为负，保证逻辑健壮性。

## 六、while vs for 选型指南（小车场景）
| 循环类型 | 核心依据 | 适用场景 | 核心提醒 |
|----------|----------|----------|----------|
| while    | 条件（True/False） | 持续加速到目标速度、人机交互调速、障碍检测 | 必须有终止条件，避免无限循环 |
| for      | 次数（range(N)） | 固定次数减速/加速、遍历速度列表 | 次数明确，无需手动终止 |

## 七、易错点总结
1. while循环未修改判断变量 → 无限循环（小车一直加速）；
2. for循环中`range(N)`计数从0开始 → 用`i+1`优化显示；
3. break/continue混淆：break终止整个循环，continue仅跳过本次；
4. 循环内未做边界检查 → 小车速度超标/过低；
5. 缩进错误：Python用缩进区分循环代码块，缩进不对则逻辑失效。
